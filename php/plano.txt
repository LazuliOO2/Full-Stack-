Fase 1 - Estrutura do Projeto

Peça ao Codex apenas:

Criar projeto Laravel + Inertia + Vue.
Configurar Tailwind.
Configurar PostgreSQL.
Criar estrutura inicial de pastas.
Não implementar regras de negócio.

Revise:

composer.json
package.json
.env.example
estrutura de diretórios

Objetivo: garantir que o projeto sobe localmente.

Fase 4 - Camada de Negócio

Aqui é onde você ganha pontos.

Peça ao Codex:

Criar Service Layer para regras de negócio.

Exemplo:

app/Services/TicketAssignmentService.php

Responsável por:

assignAutomatically()

Regra:

Escolher agente ativo
com menor quantidade
de chamados abertos.

Evite colocar essa lógica no Controller.

Isso mostra separação de responsabilidades.

Fase 5 - Segurança

Aqui você quer brilhar.

Mesmo que o desafio não peça autenticação.

Validação

Criar Form Requests:

StoreTicketRequest
UpdateTicketRequest

Exemplo:

'title' => ['required','string','max:255']

Nunca validar dentro do controller.

Mass Assignment

Models:

protected $fillable = [...]

ou

protected $guarded = [];

de forma consciente.

Sanitização

Remover:

<script>

da descrição.

Pode usar:

strip_tags()

ou uma biblioteca apropriada.

Rate Limiting

Mesmo sendo simples:

ThrottleRequests

na criação de chamados.

Mostra preocupação com abuso.

CSRF

Garantir proteção padrão Laravel.

Documentar no README.

SQL Injection

Usar apenas:

Eloquent

ou

Query Builder

Nunca SQL concatenado.

Logs

Criar logs para:

Chamado criado
Chamado atualizado
Responsável alterado

Utilizando:

Log::info()
